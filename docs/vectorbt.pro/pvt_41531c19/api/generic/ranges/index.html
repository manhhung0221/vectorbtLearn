    ranges module¶

    Module for working with range records.

    Range records capture information on time intervals. They are useful for analyzing durations of processes, such as drawdowns, trades, and positions, as well as evaluating time between events (e.g., entry and exit signals).

    Each range has a starting and an ending point. For example, range(20) yields a start of 0 and an end of 20 (not 19).

    Note

    If a range remains open in a column, its end_idx points to the latest index. Ensure you account for this when calculating custom duration metrics.

    >>> from vectorbtpro import *

>>> start = '2019-01-01 UTC'  # crypto is in UTC
>>> end = '2020-01-01 UTC'
>>> price = vbt.YFData.pull('BTC-USD', start=start, end=end).get('Close')

    100%

    >>> fast_ma = vbt.MA.run(price, 10)
>>> slow_ma = vbt.MA.run(price, 50)
>>> fast_below_slow = fast_ma.ma_above(slow_ma)

>>> ranges = vbt.Ranges.from_array(fast_below_slow, wrapper_kwargs=dict(freq='d'))

>>> ranges.readable
   Range Id  Column           Start Timestamp             End Timestamp  \
0         0       0 2019-02-19 00:00:00+00:00 2019-07-25 00:00:00+00:00
1         1       0 2019-08-08 00:00:00+00:00 2019-08-19 00:00:00+00:00
2         2       0 2019-11-01 00:00:00+00:00 2019-11-20 00:00:00+00:00

   Status
0  Closed
1  Closed
2  Closed

>>> ranges.duration.max(wrap_kwargs=dict(to_timedelta=True))
Timedelta('156 days 00:00:00')

    From accessors¶

    All generic accessors have a ranges property and a get_ranges method:

    >>> # vectorbtpro.generic.accessors.GenericAccessor.ranges.coverage
>>> fast_below_slow.vbt.ranges.coverage
0.5081967213114754

    distance_measure field¶

    Method for measuring distance (e.g., MAE, MSE, RMSE).

    See DistanceMeasure.

    error_type field¶

    Error computation mode.

    See ErrorType.

    interp_mode field¶

    Interpolation mode for mapping array values.

    See InterpMode.

    invert field¶

    Invert the pattern by reflecting its values.

    max_error field¶

    Maximum error threshold for normalization.

    If provided as an array, it must match the size of the pattern and be on the same scale.

    max_error_as_maxdist field¶

    Indicates whether PSC.max_error represents the maximum distance at each point.

    If False, exceeding PSC.max_error sets the distance to the maximum derived from PSC.pmin, PSC.pmax, and the pattern value at that point. If True and any point in a window is NaN, that point is skipped.

    max_error_interp_mode field¶

    Interpolation mode for PSC.max_error.

    If None, defaults to PSC.interp_mode.

    See InterpMode.

    max_error_strict field¶

    If True, any instance of exceeding PSC.max_error results in a similarity of NaN.

    max_pct_change field¶

    Maximum percentage change allowed for a window to remain a search candidate.

    Window similarity is set to NaN if this threshold is exceeded.

    max_records field¶

    Maximum number of records to be filled.

    If None, defaults to the number of rows in the source array.

    max_window field¶

    Maximum length of the rolling window for matching.

    If None, defaults to PSC.window.

    min_pct_change field¶

    Minimum percentage change required for a window to remain a search candidate.

    Window similarity is set to NaN if this threshold is not met.

    min_similarity field¶

    Minimum similarity threshold.

    If the computed similarity falls below this, returns NaN.

    minp field¶

    Minimum number of observations in the price window required to yield a value.

    name field¶

    Optional name assigned to the configuration.

    overlap_mode field¶

    Mode for handling overlapping matches.

    See OverlapMode.

    pattern field¶

    Flexible array representing the pattern to locate.

    Can be smaller or larger than the source array. In such cases, the smaller array is stretched using the interpolation mode specified by PSC.interp_mode.

    pmax field¶

    Maximum value used for rescaling PSC.pattern.

    Used in rescaling with RescaleMode.MinMax and for computing the maximum distance at each point when PSC.max_error_as_maxdist is disabled.

    If set to NaN, it is calculated dynamically.

    pmin field¶

    Minimum value used for rescaling PSC.pattern.

    Used in rescaling with RescaleMode.MinMax and for computing the maximum distance at each point when PSC.max_error_as_maxdist is disabled.

    If set to NaN, it is calculated dynamically.

    rescale_mode field¶

    Rescaling mode for adjusting the ranges of the input array and PSC.pattern.

    See RescaleMode.

    roll_forward field¶

    Process windows in forward direction if True; otherwise, in backward direction.

    row_select_prob field¶

    Probability of selecting a row.

    vmax field¶

    Maximum value used for rescaling the input array.

    Use only when the array has fixed bounds. Used in rescaling with RescaleMode.MinMax and for verifying PSC.min_pct_change and PSC.max_pct_change.

    If set to NaN, it is calculated dynamically.

    vmin field¶

    Minimum value used for rescaling the input array.

    Use only when the array has fixed bounds. Used in rescaling with RescaleMode.MinMax and for verifying PSC.min_pct_change and PSC.max_pct_change.

    If set to NaN, it is calculated dynamically.

    window field¶

    Base length of the rolling window for matching.

    If None, defaults to the length of PSC.pattern.

    window_select_prob field¶

    Probability of selecting a window size.

    Build PatternRanges from all occurrences of a pattern in an array.

    This class method searches for parameters of type Param among the provided arguments. If such parameters are found, they are broadcasted and combined via combine_params into one or more search configurations. If no Param arguments are detected, a single search configuration is built from the given parameters. When search_configs is supplied, it is used directly without modification. For example, passing min_similarity of 95% will use it in all search configurations except where it was explicitly overridden.

    Each search configuration is resolved using PatternRanges.resolve_search_config. The configurations are executed with execute, the resulting records arrays are concatenated, and the outcome is wrapped in a PatternRanges instance.

    Args

    Generate projections for each range record.

    This method generates projections based on range records and provided close price data.

    Args

    close : Optional[ArrayLike]

    Array of close price values.

    If None, uses Ranges.close.

    proj_start : Union[None, str, int, FrequencyLike]

    Defines when to start the projection.

    Use an integer for a specified number of rows after the start row or a timedelta-like value. The second option requires the index to be datetime-like, or at least the frequency to be set. The conversion uses to_timedelta64.

    proj_period : Union[None, str, int, FrequencyLike]

    Defines the projection length.

    Use an integer for row count or a timedelta-like value.

    incl_end_idx : bool
    Whether the end index of a range is inclusive.
    extend : bool

    Whether to extend the projection to a fixed length (beyond the end).

    The extension period is taken from the longest range duration if proj_period is None, and from the longest proj_period if not None.

    rebase : bool

    Rebase projection values so that each projection starts with start_value.

    If False, the projection retains the original close values.

    start_value : ArrayLike

    Initial value for rebasing.

    Can be a scalar or an array per column. If set to -1, uses the latest close value.

    ffill : bool
    Forward fill NaN values in the projection, even if they are NaN in close.
    remove_empty : bool
    Remove projections that are NaN or contain only a single element.
    return_raw : bool
    Return the raw output (range indices and projections) instead of a wrapped DataFrame.
    start_index : Optional[Timestamp]
    Starting timestamp to define the DataFrame index of the projection.
    id_level : Union[None, str, IndexLike]

    Identifier or key for naming range IDs.

    If a string, it may refer to a field mapping.

    jitted : JittedOption

    Option to control JIT compilation.

    See resolve_jitted_option.

    wrap_kwargs : KwargsLike

    Keyword arguments for wrapping the result.

    See ArrayWrapper.wrap.

    clean_index_kwargs : KwargsLike

    Keyword arguments for cleaning MultiIndex levels.

    See clean_index.

    Returns

    Union[Tuple[Array1d, Array2d], Frame]
    If return_raw is True, returns a tuple with a 1D array of record indices and a 2D array of projections, with each row corresponding to a range; otherwise, returns a DataFrame with projections.

    As opposed to the Numba-compiled function, the returned DataFrame has projections stacked along columns rather than rows. Set return_raw to True to obtain the original format.
